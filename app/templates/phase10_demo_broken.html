What This Page Is For

This demo page helps stakeholders understand the Phase 10 PoC validation scope and what has been prepared. Phase 10 is the live validation gate where the system will be tested against the client's real OneDrive/Excel environment.

What Is Prepared All Phase 10 read-from-Excel logic is implemented and internally tested. The system is ready for live validation.
What Is Needed Client credentials and access to the real OneDrive environment are required to execute the live PoC validation.
What Will Be Validated Authentication, OneDrive access, workbook resolution, data reading, and end-to-end HQ view from live Excel.
Demo Mode vs Live Mode
Demo Mode ON: Shows sample data to visualize how the system works. This is for demonstration purposes only — it does not prove real environment connectivity.
Demo Mode OFF: Calls real API endpoints. When client credentials are configured, this will show actual OneDrive Excel data.

Phase 10 Validation Checklist

These are the checks to be performed when credentials arrive. Items marked Prepared have implementation ready; items marked Pending require live environment access.

  • MSAL Authentication Flow OAuth2 client credentials flow implemented and tested with mock tokens
  • Real Token Acquisition Pending: Validate token acquisition with client's Azure AD credentials
  • OneDrive Path Resolution Location/staff folder path logic implemented with configurable base paths
  • Live Folder Access Pending: Verify access to client's actual OneDrive folder structure
  • Excel Workbook Reading Graph API workbook/worksheet/range read logic fully implemented
  • Live Data Read Pending: Read actual receipt data from client's Excel files
  • Error Handling & Retry Implemented: Rate limiting, token refresh, network retry with exponential backoff
  • End-to-End HQ View Pending: Full validation that HQ dashboard shows live Excel data correctly

Phase Scope Summary

Phase 10: Read from Excel (PoC Validation Gate)

HQ reads receipt data directly from OneDrive Excel files via Graph API. This is the primary validation target.

Internal 11A: Write to Excel (Available but not primary focus)

Graph API writers implemented for direct OneDrive Excel writes. Internal use only until Phase 10 is validated.

Internal 11B: Reconciliation (Available but not primary focus)

Draft-Excel conflict detection implemented. Becomes relevant after write operations are live.
